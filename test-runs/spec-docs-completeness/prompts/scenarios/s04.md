# S04 - Implementation Update

## Prompt

In the s04 fixture directory, `docs/spec-docs/` exists with existing specs for a payments module. The code has been updated: a `refundPayment` function was added to `src/payments.js`. The function accepts a `paymentId` and returns `{ refunded: true, paymentId }`.

Run `spec-docs update` to sync specs after this implementation change.

## Checklist

Verify the update output:

- [ ] Classifies the change as Level 2 because `refundPayment` adds localized payments behavior and updates one module spec plus targeted inventory symbols.
- [ ] Updates the payments spec (e.g., `specs/modules/payments.spec.md`) to include `refundPayment` behavior.
- [ ] Adds `refundPayment` facts evidenced by the fixture: function signature (`paymentId` parameter) and return value (`{ refunded: true, paymentId }`). Does not invent input validation, state validation, or gateway call behavior absent from the fixture code.
- [ ] Adds `refundPayment` to the implementation mapping in the payments spec (key symbols). Includes call flow and data flow only if present in the fixture; does not invent behavior absent from the fixture code.
- [ ] Updates frontmatter on the payments spec: `verified_commit`, `verified_date`, `source_files`, `symbols` as needed.
- [ ] Updates `inventory.md` Symbol-to-Spec Index to include `refundPayment`.
- [ ] Updates Boundary and/or Failure Localization sections ONLY if `refundPayment` introduces new boundary concerns or failure modes that were not previously covered. Does not add speculative boundary/failure text.
- [ ] Does NOT perform an unrelated rewrite of the entire payments spec.
- [ ] Does NOT silently rewrite architecture rules in `docs/spec-docs/architecture/`.
- [ ] Does NOT rewrite specs for modules that were not changed.
- [ ] Runs targeted light check with `Scope: targeted-check`.
- [ ] Does NOT claim full `verify` PASS for this Level 2 update.
- [ ] If targeted light check reports findings, those findings are about the update scope, not pre-existing issues unrelated to `refundPayment`.

Result: PASS / PASS_WITH_NOTES / FAIL / BLOCKED
