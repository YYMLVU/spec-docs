# S17 - Level 1 Single-Spec Touch

## Prompt

In the fixture project, one private edge-case behavior changed inside an existing mapped parser module. The change affects only `specs/modules/parser.spec.md`. No public contract, inventory mapping, source path, exported symbol, architecture rule, or ADR changed.

Run `spec-docs update` and classify the impact.

## Checklist

Verify the update output:

- [ ] Classifies the change as Level 1.
- [ ] Reads and updates only `specs/modules/parser.spec.md`.
- [ ] Updates only the affected behavior, edge case, verification point, or evidence.
- [ ] Does not update unrelated specs.
- [ ] Does not edit `inventory.md`.
- [ ] Does not run full verify.
- [ ] States why Level 2 or higher was not required.

Result: PASS / PASS_WITH_NOTES / FAIL / BLOCKED
