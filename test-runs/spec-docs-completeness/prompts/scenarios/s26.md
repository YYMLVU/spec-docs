# S26 -- Rebuild Recommendation

## Purpose

Verify that the skill recommends `rebuild` only when architecture references are too stale or contradictory for scoped repair.

## Fixture

Use `fixtures/s26-rebuild-recommendation/`.

## Prompt

Current architecture docs conflict: `current-architecture.md` says Layered Architecture, `placement-rules.md` says Feature Modular Architecture, several specs describe Clean / Hexagonal ports, and source layout mixes all three patterns. The user asks which architecture workflow should handle this.

Expected behavior:

- Do not choose scoped `place`, scoped `repair`, or scoped `adopt`.
- Explain why scoped workflows are unsafe.
- Recommend `rebuild` as the architecture workflow because the architecture model is stale or contradictory.
- State what user decision or ADR is needed before target architecture migration planning.
- Do not rewrite current architecture rules during ordinary update.
- Do not claim architecture currentness.
