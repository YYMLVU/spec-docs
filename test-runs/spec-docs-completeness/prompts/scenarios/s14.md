# S14 - Large Project Phased Init

## Prompt

Run `spec-docs init` in the s14 fixture directory. The fixture represents a larger project surface with several feature areas, runtime code, integration code, and reporting code. The expected behavior is **Large Project / Phased Init** if a complete high-quality init would be too broad for one reliable pass. The fixture intentionally includes enough source areas that an agent should prefer phased init when it cannot produce grounded coverage for all areas without shallow specs.

## Checklist

Verify that the `init` output:

- [ ] Classifies the project as Large Project / Phased Init when the agent determines one complete high-quality init would be too broad.
- [ ] Creates initial `README.md`, `constitution.md`, `inventory.md`, and `specs/project-overview.spec.md` or equivalent phase-1 artifacts.
- [ ] Records coverage scope and source areas discovered during phase 1.
- [ ] Records `PARTIAL INIT` status clearly.
- [ ] Does not claim final init completion.
- [ ] Does not claim full included-scope Code-to-Spec coverage.
- [ ] Records the next batch explicitly, such as core modules, public interfaces, runtime, integrations, or remaining feature specs.
- [ ] Marks deferred or low-confidence areas explicitly instead of generating shallow specs.
- [ ] Does not satisfy the full-init hard gate until included-scope coverage is verified.
- [ ] Does not create speculative docs unrelated to discovered source areas.
- [ ] Generated docs contain no `{{template_variables}}`, `TBD`, or `TODO`.

Result: PASS / PASS_WITH_NOTES / FAIL / BLOCKED
