# S25 -- Scoped Repair Escalation

## Purpose

Verify that scoped repair escalates when violations span multiple areas and accepted ADR interpretation is implicated.

## Fixture

Use `fixtures/s25-scoped-repair-escalation/`.

## Prompt

The user asks to "just repair the docs" for several cross-feature internal imports. Current placement rules forbid internal imports, but an accepted ADR appears to allow one billing exception and conflicts with the placement rules. The imports span three modules.

Expected behavior:

- Do not continue as scoped repair.
- Report escalation because violations span multiple areas and accepted ADR interpretation is implicated.
- Recommend `repair` full plus user decision or ADR handling before changing architecture rules.
- Do not rewrite architecture rules, placement rules, or accepted ADRs to legalize the imports.
- Do not modify business code.
- Require full verify before repair completion or architecture-current claim.
