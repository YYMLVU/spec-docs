# S03 - Placement Before Feature Work

## Prompt

In the s03 fixture directory, `docs/spec-docs/` already exists and has been initialized. The project uses Feature Modular Architecture.

Run `spec-docs place` for a "billing reminder" feature. The feature intent is: send scheduled billing reminder emails to users with upcoming due dates. This feature involves a new schedule module, email delivery integration, and possibly touching billing data.

## Checklist

Verify the placement review output:

- [ ] Output follows the Placement & Boundary Review format (feature intent, ownership, layer placement, boundary contract, etc.).
- [ ] Consumes the Feature Modular Architecture preset -- checks owner feature/module and forbids imports into another feature's internal implementation.
- [ ] Identifies an owning module for the billing reminder feature.
- [ ] Lists allowed dependencies (which modules this feature can depend on).
- [ ] Lists forbidden dependencies (modules this feature must not internally import from).
- [ ] Specifically checks and forbids internal cross-feature access (e.g., billing-reminder must not import internal implementation from billing, or vice versa).
- [ ] Defines required public contracts if cross-feature data or events are needed.
- [ ] If ownership of any aspect (e.g., "does the schedule live inside billing-reminder, billing, or a shared scheduling module?") is unclear, outputs `Needs ADR` or `Needs User Decision`.
- [ ] Lists specs and architecture files to read before implementation.
- [ ] Lists implementation constraints derived from architecture rules.
- [ ] Provides failure localization hints for the new feature.
- [ ] Lists specs to update after implementation.
- [ ] Records open questions with specific `[NEEDS CLARIFICATION: ...]` markers where information is missing.
- [ ] Does not modify any code.
- [ ] Does not create implementation specs as if the behavior already exists.
- [ ] Confidence level is stated (e.g., "High", "Medium", "Low", or "Tentative").
- [ ] Decision rationale is recorded.

Result: PASS / PASS_WITH_NOTES / FAIL / BLOCKED
