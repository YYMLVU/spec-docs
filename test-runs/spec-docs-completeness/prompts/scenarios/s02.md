# S02 - Existing Small App Init

## Prompt

Run `spec-docs init` in the s02 fixture directory. The fixture contains a small existing application with implementation-relevant files. The application is a simple CLI notes manager with source files `src/cli.js` (entrypoint, argument parsing, command dispatch) and `src/notes.js` (note storage with add/list operations using in-memory array storage). There may be minimal tests and a package.json.

## Checklist

Verify that the `init` output:

- [ ] Reads and grounds facts in `src/cli.js` -- identifies the CLI entrypoint, argument parsing, command dispatch, and how it maps to spec content.
- [ ] Reads and grounds facts in `src/notes.js` -- identifies note storage mechanism (in-memory array), add/list operations only, and data format.
- [ ] Covers CLI responsibilities: argument parsing, command routing, output formatting.
- [ ] Covers notes module responsibilities: in-memory storage, add/list operations, data model.
- [ ] Identifies the tech stack from actual code (Node.js, relevant packages from package.json).
- [ ] Creates `docs/spec-docs/inventory.md` with real included/excluded globs, Spec List, Code-to-Spec Index, and Symbol-to-Spec Index.
- [ ] Creates `docs/spec-docs/specs/project-overview.spec.md` with actual implementation facts, not empty-project baseline wording.
- [ ] Creates relevant child specs (e.g., `modules/cli.spec.md`, `modules/notes.spec.md`) grounded in actual source files.
- [ ] Implementation mapping in each spec includes key symbols, call flow, data flow, and code references.
- [ ] Architecture is identified from actual code patterns -- does not invent an architecture that contradicts the code.
- [ ] If architecture cannot be clearly determined from the small codebase, records `[NEEDS CLARIFICATION]` rather than guessing.
- [ ] No roadmap, backlog, tasks.md, or plan.md exists anywhere in `docs/spec-docs/`.
- [ ] `constitution.md` is created with durable principles, not implementation-specific details.
- [ ] Protocol block is installed in `AGENTS.md` and/or `CLAUDE.md` exactly once.
- [ ] `verify` passes or correctly reports findings before `init` declares completion.
- [ ] Generated docs contain no `{{template_variables}}`, `TBD`, or `TODO` placeholders.
- [ ] Notes missing architecture -- if the codebase is too small to determine architecture selection confidently, the skill does not guess a Primary Preset but records a `[NEEDS CLARIFICATION]` marker.

Result: PASS / PASS_WITH_NOTES / FAIL / BLOCKED
