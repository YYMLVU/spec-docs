# S02 - Minimal Existing Project Init

## Prompt

Run `spec-docs init` in the s02 fixture directory. The fixture contains a minimal existing application with implementation-relevant files. The application is a simple CLI notes manager with source files `src/cli.js` and `src/notes.js`, plus a minimal `package.json`.

The expected project profile is **Minimal Existing Project** because the project has only a few implementation files, one small CLI entrypoint, and no meaningful architecture or module ownership boundaries beyond the two-file implementation.

## Checklist

Verify that the `init` output:

- [ ] Classifies the project as Minimal Existing Project, not Empty Project and not Standard Existing Project.
- [ ] Reads and grounds facts in `src/cli.js` -- identifies the CLI entrypoint, argument parsing, command dispatch, and output behavior.
- [ ] Reads and grounds facts in `src/notes.js` -- identifies note storage mechanism, add/list operations only, and data format.
- [ ] Identifies the tech stack from actual code and `package.json`.
- [ ] Creates `docs/spec-docs/README.md`.
- [ ] Creates `docs/spec-docs/constitution.md`.
- [ ] Creates `docs/spec-docs/inventory.md` with real included/excluded globs and minimal Code-to-Spec mapping.
- [ ] Creates `docs/spec-docs/specs/project-overview.spec.md` with actual implementation facts.
- [ ] Does not create `docs/spec-docs/architecture/` by default.
- [ ] Does not create `docs/spec-docs/decisions/` by default.
- [ ] Does not create `docs/spec-docs/rebuild/` by default.
- [ ] Does not create empty category folders such as `features/`, `modules/`, `interfaces/`, `runtime/`, `data/`, `integrations/`, or `quality/` unless the output includes a grounded reason from code.
- [ ] Does not create separate child specs such as `modules/cli.spec.md` or `modules/notes.spec.md` by default; these responsibilities can be summarized in the project overview for this minimal project.
- [ ] Records architecture as not applicable or not enough evidence instead of inventing a Primary Preset.
- [ ] No roadmap, backlog, tasks.md, or plan.md exists anywhere in `docs/spec-docs/`.
- [ ] Protocol block is installed in `AGENTS.md` and/or `CLAUDE.md` exactly once.
- [ ] Minimal-profile verify passes or correctly reports warnings before `init` declares completion.
- [ ] Generated docs contain no `{{template_variables}}`, `TBD`, or `TODO`.

Result: PASS / PASS_WITH_NOTES / FAIL / BLOCKED
