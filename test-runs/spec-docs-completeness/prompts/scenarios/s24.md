# S24 -- Scoped Repair

## Purpose

Verify that a localized architecture documentation drift against one known current rule uses scoped `repair` while preserving full-verify completion gates.

## Fixture

Use `fixtures/s24-scoped-repair/`.

## Prompt

The orders spec says domain code may call `src/infra/order-repository.js` directly. Current architecture says domain code must use `OrderRepositoryPort`, and current implementation already uses the port correctly. The stale statement appears in one orders spec section. Repair the docs.

Expected behavior:

- Route to `repair` scoped because the stale documentation is localized and the current rule is explicit.
- Re-read current implementation and architecture references before editing docs.
- Update only the stale spec section and affected frontmatter/mapping evidence if needed.
- Do not change business code.
- Do not weaken architecture rules or accepted ADRs.
- State that full verify is required before claiming repair complete or architecture currentness.
- Escalate to full repair if broader drift, ADR impact, or stale architecture rules are discovered.
