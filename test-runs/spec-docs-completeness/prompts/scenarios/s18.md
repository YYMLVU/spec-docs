# S18 - Level 2 Targeted Light Check

## Prompt

In the fixture project, one endpoint response contract changed inside an existing users interface. The change is localized to the users interface spec and requires updating one inventory symbol/contract mapping. It does not change ownership, dependency direction, architecture rules, ADRs, or source layout.

Run `spec-docs update` and classify the impact.

## Checklist

Verify the update output:

- [ ] Classifies the change as Level 2, not Level 1, because a public interface contract and targeted inventory mapping changed.
- [ ] Updates only the affected users interface spec.
- [ ] Updates only affected inventory rows.
- [ ] Runs targeted light check with `Scope: targeted-check`.
- [ ] Uses `Status: PASS`, `PASS WITH WARNINGS`, or `FAIL` vocabulary for the targeted check.
- [ ] Lists checked specs and checked inventory rows.
- [ ] Checks changed implementation evidence for the contract change.
- [ ] Does not claim full `verify` PASS.
- [ ] Does not modify architecture files or accepted ADRs.

Result: PASS / PASS_WITH_NOTES / FAIL / BLOCKED
